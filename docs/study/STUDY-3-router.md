# 勉強ノート3: 自分のルーターの仕組み — 今のルールから学習器まで

あなたが考えた「簡単ならローカル、難しければクラウド」の中身を、
機械学習の言葉に翻訳する。ここが分かると、将来 `router.py` を学習器に
差し替える工程の全ステップが「知っている話」になる。

## 1. ルーターの正体は「二値分類器」

ルーターの仕事を一文にすると:
**「このプロンプトはローカルで成功するか? Yes / No を予測する」**

入力(プロンプト)から Yes/No を予測する装置を**二値分類器**と呼ぶ。
迷惑メールフィルタ(スパムか否か)と完全に同じ問題形式。
つまりあなたが作ろうとしているのは、スパムフィルタの親戚である。

## 2. 間違いには2種類あり、痛さが違う(ここが設計の核心)

| 間違い | 何が起きるか | 痛さ |
|---|---|---|
| ローカルで足りるのにクラウドへ | 枠・$を無駄にする | 軽い(結果は正しい) |
| ローカルでは無理なのにローカルへ | 失敗 or 時間を溶かして結局昇格 | 重い |

この**非対称性**が「迷ったらクラウド寄せ」(`router.py` のコメント)の正体。
MLの言葉では、2種類の間違いのバランスを取る指標を**適合率(precision)と再現率(recall)**
と呼ぶ。がん検診で「見逃し(重い)」と「誤検知(軽い)」を区別するのと同じ考え方。

覚え方: **間違いをどちらに倒すかは、痛い方を避ける向きに倒す**。

## 3. 今のルーター(rule-v1)をMLの言葉で読む

```python
if len(prompt) < 120 and any(h in prompt for h in SIMPLE_HINTS):
    return ローカル
return クラウド
```

これは「**手書きの特徴量 + 手書きのしきい値**」による分類器と読める:
- 特徴量1: プロンプトの長さ(120字未満か)
- 特徴量2: 簡単キーワードを含むか
- 判定: 両方 Yes ならローカル

ML化とは、この「手書き」の部分をデータに決めさせること。**構造は同じ**。
だから今のルールは捨てるものではなく、学習器の「特徴量の第一候補リスト」になる。

## 4. 学習ルーター第1段: ロジスティック回帰 = 重み付きチェックリスト

ロジスティック回帰は名前がいかついが、実体は**採点表**:

```
プロンプトが短い        → +3点
「rename」を含む        → +2点
seed ファイルが1個      → +1点
「設計」「リファクタ」を含む → -4点
--------------------------------
合計点を 0〜1 の確率に変換 → 0.5 以上ならローカル
```

学習とは、**この点数(重み)を過去データから自動で決める**こと。
「ローカルで成功した例では何が多かったか / 失敗した例では何が多かったか」を
数えて、効く特徴に大きい点、効かない特徴にほぼ0点を割り当てる。

- 教師データ: DESIGN-dataset の routing ペア(特徴量 + `local_success` ラベル)
- 必要量の目安: 100ペア〜。特徴量が10個程度なら十分戦える
- 高卒数学で足りるか: **足りる**。中身は掛け算と足し算。確率への変換だけ
  滑らかな変換関数(シグモイド)を1つ使うが、「合計点を0〜1に潰すゴムシート」と
  思えばよい

## 5. しきい値は「性格のつまみ」

学習器は確率(例: ローカル成功率 0.72)を出す。どこから上をローカルに送るかの
**しきい値**は学習後に人間が選ぶ調整つまみ:

- しきい値 0.9: 慎重派。ローカルにはほぼ確実な物だけ → 節約は少ないが失敗も少ない
- しきい値 0.5: 強気派。節約は増えるが失敗(昇格のやり直し)も増える

§2 の「間違いの痛さの非対称」をこのつまみ1つで表現できる。
将来 quota-aware routing(R3)をやるなら、「枠が残り少ない日はしきい値を上げる」
という動的なつまみ操作になる。設計が一本につながっている。

## 6. 罠: ルーターの成績はどう測るか

**罠1 — 精度(accuracy)は当てにならない。**
もしタスクの9割がローカルで解けるなら、「全部ローカル」と言うだけの壊れた分類器でも
精度90%になる。だから2種類の間違いを別々に数える(§2 の precision / recall)。

**罠2 — タスク単位で訓練/テストを分ける。**
同じタスクの反復1を訓練に、反復2をテストに入れると、分類器はタスクを暗記して
高得点を出す(=リーク。STUDY-2 §1 と同じ病気の別の顔)。
評価は必ず**見たことのないタスク**で行う。

**罠3 — 較正(calibration)。**
「確率 0.7 と言った物が本当に7割成功しているか」。自信過剰な分類器は
しきい値のつまみが効かなくなる。R4 の「小型モデルは自分ができないことを
知っているか」も同じ較正の話。

## 7. その先の2つの進化(バックログとの接続)

- **カスケード(R1)**: 予測をやめて「まずローカルにやらせ、ダメなら昇格」。
  分類の間違いを実行結果で回収する方式。予測型とどちらが得かは
  「ローカルの失敗1回の時間コスト」次第 — これも計測で答えが出る問い。
- **バンディット(R2後半)**: 常に予測通りに送ると、「ローカルに送らなかったタスクを
  ローカルが解けたか」のデータが二度と貯まらない。だから**たまにわざと逆に送る**
  (探索)。「活用(今の最善)と探索(将来のための実験)のバランス」は
  強化学習の入口の概念で、あなたのプロジェクトで最初に出会う場所がここ。
